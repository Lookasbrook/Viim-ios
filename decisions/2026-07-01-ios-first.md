# ADR — MVP iOS d'abord, malgré 92% de parts Android au Burkina

**Date** : 2026-07-01 · **Statut** : Accepté · **Décideur** : Guy (Teli)

## Contexte
Android domine à 92% au Burkina Faso. Un MVP Android semblerait logique commercialement.

## Décision
Le MVP est développé en **Swift natif iOS 16+**.

## Justification
Les testeurs initiaux (groupe fermé famille/amis) sont sur iPhone ; la phase actuelle valide la **fiabilité des capteurs**, pas le marché. Le compte développeur Apple est déjà configuré. Les APIs capteurs iOS (CoreMotion/CoreLocation) sont homogènes entre appareils, ce qui isole les variables pendant la calibration.

## Conséquences
- Les algorithmes (filtrage, seuils, score) sont documentés de façon agnostique dans `architecture/sensor-algorithms.md` pour faciliter le portage Android (Kotlin) après validation.
- Aucune dépendance iOS-only dans le backend.
- Le portage Android est la priorité n°1 post-validation MVP.
